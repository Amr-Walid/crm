# Project Rules

- **Automatic Commits**: After making any changes or edits to files in this project, you must automatically create a Git commit with a clear, concise commit message describing the changes made. Do not wait for the user to ask for a commit.

- **Do Not Commit Ignored Files**: Never force-add or commit any files that are listed in the `.gitignore` file.

- **Review Files Before Answering**: Always view and inspect the contents of target files before responding to the user's questions or request changes, ensuring the answer is fully accurate and tailored to the actual implementation.

- **Use Context7 MCP for Documentation**: Whenever working with or asking about libraries, frameworks, SDKs, APIs, or tools (including .NET 9, EF Core 9, Microsoft.Extensions.Caching.Hybrid, Redis, Docker, and Chatwoot APIs), always use the Context7 MCP server (`resolve-library-id` followed by `query-docs`) to retrieve the most up-to-date documentation. Avoid relying on training data or generic web searches for library-specific information.
